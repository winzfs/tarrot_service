# 06. 리딩 흐름과 배열법 공통 시스템

이 문서는 `tarrot_service`에서 타로 배열법을 확장하거나 유지보수할 때 지켜야 할 **공통 연출 흐름, UI 구조, 해석 방식, 터치 규칙**을 정의한다.

목표는 카드 수만 늘리는 것이 아니다.

> 어떤 배열법을 사용하더라도, 사용자가 같은 점술관 안에서 같은 의식의 규칙을 경험하게 만든다.

## 1. 핵심 원칙

실제 타로 리딩은 카드 하나의 뜻만 읽지 않는다.

```txt
질문
  ↓
배열법 선택
  ↓
카드 위치 의미
  ↓
카드 자체의 의미
  ↓
카드 간 관계
  ↓
전체 흐름
  ↓
현실적인 조언
```

게임에서도 이 순서를 유지한다.

현재 구현은 1장/3장/5장 배열을 지원한다. 배열 추천은 룰 기반 임시 추천과 AI 기반 추천을 함께 사용한다. 정방향/역방향은 아직 본격 구현 전이며, 다음 확장 후보로 남겨둔다.

## 2. 현재 기본 흐름

```txt
IntroScene
  ↓
QuestionScene
  ↓
질문 카테고리와 질문 입력
  ↓
룰 기반 임시 배열 추천 즉시 표시
  ↓
/api/spread-recommendation AI 배열 추천
  ↓
점술사가 고른 배열 표시
  ↓
사용자 수동 배열 변경 가능
  ↓
질문 봉인 연출
  ↓
CardSelectScene
  ↓
배열법에 맞는 카드 수만큼 카드 공개
  ↓
ReadingScene
  ↓
카드별 챕터 해석
  ↓
종장 종합 조언
  ↓
SummaryScene
  ↓
별빛의 기록 요약 / 공유 / 이미지 저장 / 텍스트 복사
```

현재 런타임의 마지막 화면은 `SummaryScene`이다. 후속 대화용 `ChatScene`은 현재 코드에서 제거된 상태이며, 다시 필요해지면 별도 확장 작업으로 복구한다.

배열법마다 달라지는 것:

```txt
- 카드 수
- 카드 위치 이름
- 카드 위치 의미
- 카드 배치 형태
- 카드별 챕터 수
- 종장 종합 방식
```

유지되는 것:

```txt
- 질문 봉인
- 사용자가 직접 카드 공개
- 카드별 챕터 해석
- 챕터/종장 터치 잠금
- 종장 종합 조언
- 리딩 결과 기록/공유
```

## 3. 현재 지원 배열법

배열법 데이터는 다음 파일에서 관리한다.

```txt
src/tarot/spreads.ts
```

현재 사용자가 직접 선택할 수 있는 기본 배열은 다음 5개다.

### 오늘의 한 장

```txt
용도: 짧은 조언, 오늘의 메시지, 가볍고 빠른 리딩
위치: 오늘의 메시지
```

### 상황과 조언의 세 문

```txt
용도: 일/돈/현실 고민, 막힘, 해결 방향이 필요한 질문
위치:
1. 현재 상황
2. 막고 있는 것
3. 조언
```

### 시간의 세 문

```txt
용도: 흐름/미래/전망 질문, 과거와 현재의 흐름이 중요한 질문
위치:
1. 과거
2. 현재
3. 미래
```

### 관계의 거울 5장

```txt
용도: 연애, 인간관계, 상대와 나의 흐름을 함께 봐야 하는 질문
위치:
1. 나의 마음
2. 상대의 흐름
3. 관계의 현재
4. 막고 있는 것
5. 조언
```

### 선택의 갈림길 5장

```txt
용도: 선택 고민, 할까 말까, 두 방향 비교
위치:
1. 선택 A
2. 선택 B
3. 숨은 변수
4. 진짜 바람
5. 선택의 조언
```

참고: `src/tarot/spreads.ts`에는 관계/선택 3장 배열도 데이터로 남아 있을 수 있다. 현재 기본 선택 UI와 AI 추천 허용 목록은 위 5개 기준으로 맞춘다.

## 4. 배열 추천 시스템

QuestionScene은 두 단계로 배열을 추천한다.

### 1단계. 룰 기반 임시 추천

목적:

- 사용자가 질문을 입력하자마자 빈 화면 없이 즉시 배열을 보여준다.
- AI 응답 전까지 기본 추천을 유지한다.
- AI 호출 실패 시 fallback으로 사용한다.

### 2단계. AI 기반 추천

```txt
질문 입력
  ↓
0.72초 debounce
  ↓
POST /api/spread-recommendation
  ↓
Worker AI가 category + question을 분석
  ↓
spreadId + reason 반환
  ↓
점술사가 고른 배열 패널 갱신
```

AI 추천은 다음 파일이 담당한다.

```txt
worker/prompts/spread.ts
worker/index.ts
src/api/client.ts
src/api/types.ts
src/game/scenes/QuestionScene.ts
```

응답 형식:

```json
{
  "spreadId": "relationship-mirror-five",
  "reason": "관계의 흐름과 막고 있는 요소를 함께 보는 것이 어울립니다."
}
```

허용되는 spreadId:

```txt
daily-one-card
situation-obstacle-advice
past-present-future
relationship-mirror-five
choice-crossroad-five
```

### 수동 선택 정책

```txt
사용자가 배열 버튼을 직접 누름
→ selectedSpreadId 설정
→ isManualSpreadSelection = true
→ AI 자동 추천 중단
→ 직접 선택한 배열 유지

질문 입력 변경
→ 수동 선택 상태면 AI 추천을 다시 호출하지 않음

카테고리 변경
→ 수동 선택 해제
→ AI 추천 상태 초기화
→ 새 카테고리/질문 기준으로 다시 추천
```

질문 화면에는 `점술사가 고른 배열` 패널을 표시한다.

표시 내용:

```txt
- 배열명
- 카드 수
- 카드 위치 구성
- 추천 이유
```

## 5. 씬별 공통 역할

### IntroScene

- 사용자를 점술관으로 초대한다.
- 카드 뒷면과 VFX로 의식의 시작을 보여준다.
- 배열법과 무관하게 공통으로 사용한다.

### QuestionScene

역할:

- 질문을 받는다.
- 질문 카테고리를 받는다.
- 룰 기반 임시 배열을 먼저 보여준다.
- AI 기반 추천 배열을 받아 화면에 반영한다.
- 사용자가 직접 배열을 바꿀 수 있게 한다.
- 질문 봉인 연출을 보여준다.

유지할 것:

```txt
- 질문은 반드시 봉인 연출을 거친다.
- 추천 배열은 숨은 로직으로만 두지 않고 화면에 보여준다.
- AI 추천 실패 시에도 룰 기반 추천으로 진행 가능해야 한다.
- 직접 배열 선택은 자동 추천보다 우선한다.
- 질문 화면 UI는 하단 브라우저/대화창 영역에 가리지 않아야 한다.
```

### CardSelectScene

역할:

- 배열법에 필요한 카드 수만큼 카드를 뽑는다.
- 카드 뒷면을 배열 위치에 배치한다.
- 사용자가 카드를 하나씩 공개한다.
- 모든 카드가 공개되면 리딩 단계로 이동한다.

현재 카드 수별 배치:

```txt
1장: 중앙 1장
3장: 가로 3장
5장: 위 2장 + 아래 3장
```

추가 기준:

```txt
- spread.positions[].layoutHint가 있으면 그 좌표를 우선 반영한다.
- layoutHint가 없으면 카드 수별 기본 배치를 사용한다.
```

### ReadingScene

역할:

- 카드별 해석을 챕터처럼 보여준다.
- 카드 위치 의미를 먼저 보여준다.
- 그 위치에 놓인 카드 의미를 질문과 연결한다.
- 마지막 종장에서 전체 흐름을 종합한다.
- 종장 이후 `SummaryScene`으로 이동한다.

현재 상태:

```txt
- 1장/3장/5장 챕터 진행 지원
- spread.positions 기반 chapterTitle/shortMeaning 사용
- 카드별 대화창 자동 표시
- 대화창 표시 후 3초 뒤 터치 가능
- 종장 카드 융합 연출
- 5장 종장 fusion-card-4, fusion-card-5 보정 적용
- 종장 이후 SummaryScene 시작
```

### SummaryScene

역할:

- 리딩 결과를 `별빛의 기록`으로 요약한다.
- 질문, 선택 카드, 배열명, 리딩 요약을 표시한다.
- 리딩 요약을 공유하거나 텍스트로 복사할 수 있게 한다.
- 리딩 결과를 이미지로 저장할 수 있게 한다.

표시/사용 데이터:

```txt
- 원래 질문
- 배열명
- 카드별 위치
- 카드명 한글/영문
- 카드별 짧은 해석 미리보기
- 리딩 title / summary
```

### ChatScene

현재 상태:

```txt
- 런타임 플로우에서 사용하지 않음
- 코드에서 제거됨
- 후속 질문 기능이 필요해질 때 별도 API와 함께 다시 설계
```

후속 대화를 되살릴 때 필요한 작업:

```txt
- src/game/scenes/ChatScene.ts 복구 또는 재작성
- GameConfig.ts scene 목록에 ChatScene 등록
- src/api/types.ts에 ChatMessage/ChatRequest/ChatResponse 추가
- src/api/client.ts에 requestChat 추가
- worker/index.ts에 /api/chat 핸들러 추가
- worker/prompts/chat.ts와 response parser 연결
- SummaryScene에서 후속 질문 진입 버튼 추가
```

## 6. 공통 리딩 시퀀스

### Step 1. 배열 추천/선택

```txt
질문 카테고리 선택
  ↓
질문 입력
  ↓
룰 기반 임시 추천
  ↓
AI 추천 요청
  ↓
추천 배열 패널 표시
  ↓
필요하면 사용자가 직접 배열 변경
```

### Step 2. 질문 봉인

```txt
- 배경 어둡게 처리
- 질문 문장이 기도문처럼 표시
- 질문이 접히거나 봉인되는 연출
- VFX 발광/링/파티클
- 선택된 배열에 맞는 카드 선택 화면으로 전환
```

### Step 3. 카드 배치

```txt
- 카드 뒷면들이 배열 위치에 등장
- 각 카드 아래 또는 주변에 위치 라벨 표시
- 사용자가 하나씩 터치해서 공개
```

### Step 4. 카드 공개

```txt
- 터치한 카드가 중심축 기준 180도 플립처럼 보임
- 공개 순간 빛 폭발
- 카드 앞면 이미지 표시
- 한글명/영문명 표시
- 공개 후 해당 카드 위치 라벨 유지
```

### Step 5. 카드별 챕터 해석

```txt
챕터 제목 등장
  ↓
위치 의미/속삭임 표시
  ↓
카드 등장
  ↓
카드명 표시
  ↓
점술사 대화창 자동 표시
  ↓
3초 후 터치 가능
  ↓
다음 카드 챕터로 이동
```

### Step 6. 종장 종합 조언

```txt
종장 제목 등장
  ↓
선택된 카드들이 빛으로 모이는 연출
  ↓
종합 조언 문장들이 한 줄씩 페이드인
  ↓
모든 문장 표시 후 3초 뒤 터치 가능
  ↓
SummaryScene으로 이동
```

### Step 7. 별빛의 기록

```txt
SummaryScene 진입
  ↓
질문/배열/카드/요약 표시
  ↓
공유, 이미지 저장, 텍스트 복사 제공
```

## 7. 터치 잠금 규칙

### 카드별 챕터

```txt
챕터 화면 진입
  → 터치 잠금
제목/카드 연출 진행
  → 터치 잠금 유지
대화창 자동 표시
  → 터치 잠금 유지
대화창 표시 후 3초 경과
  → 터치 가능
터치
  → 다음 챕터 또는 종장
```

### 종장

```txt
종장 화면 진입
  → 터치 잠금
카드 융합/본문 페이드인
  → 터치 잠금 유지
모든 문장 표시 후 3초 경과
  → 터치 가능
터치
  → SummaryScene
```

## 8. 배열법 데이터 구조

```ts
export type TarotSpread = {
  id: string;
  name: string;
  subtitle: string;
  description: string;
  recommendedFor: ReadingCategory[];
  cardsToDraw: number;
  positions: TarotSpreadPosition[];
};

export type TarotSpreadPosition = {
  id: string;
  label: string;
  chapterTitle: string;
  shortMeaning: string;
  promptMeaning: string;
  aura?: "past-aura" | "present-aura" | "future-aura";
  layoutHint?: {
    x: number;
    y: number;
  };
};
```

AI 해석을 위해 반드시 필요한 정보:

```txt
- position id
- 화면 표시용 위치 이름
- 챕터 제목
- 짧은 의미
- 프롬프트용 상세 의미
```

## 9. AI 해석 순서

```txt
1. 이 카드는 어떤 위치에 놓였는가?
2. 그 위치는 질문에서 무엇을 의미하는가?
3. 이 카드의 기본 상징은 무엇인가?
4. 사용자의 질문과 연결하면 어떤 의미인가?
5. 카드 간 흐름 속에서 어떤 역할인가?
6. 사용자가 지금 취할 수 있는 현실적 태도는 무엇인가?
```

정방향/역방향은 아직 구현 전이다.

## 10. 카드 간 관계 해석

종장에서는 반드시 카드 간 관계를 본다.

```txt
- 메이저 아르카나가 많은가?
- 같은 수트가 반복되는가?
- 숫자 흐름이 이어지는가?
- 특정 카드가 전체 분위기를 압도하는가?
- 관계형 배열에서는 나와 상대의 카드가 충돌하는가, 보완하는가?
- 선택형 배열에서는 두 선택의 에너지가 어떻게 다른가?
- 5장 배열에서는 장애물과 조언 카드가 앞선 카드들을 어떻게 정리하는가?
```

## 11. AI 응답 길이 기준

```txt
1장
- 카드 해석 2~3문장
- advice 2문장

3장
- 카드별 2~3문장
- advice 3~4문장

5장
- 카드별 2문장 중심, 최대 3문장
- advice 4~5문장
- advice에서 카드 하나씩 재나열하지 말고 전체 구조와 카드 간 관계를 종합
```

## 12. UI 재사용 원칙

새 배열법을 만들 때 새 화면을 무조건 만들지 않는다.

우선 재사용할 것:

```txt
- 질문 봉인 연출
- 카드 뒷면 표시
- 카드 공개 플립
- 카드 앞면 표시
- 카드명 한글/영문 표시
- 챕터 제목 연출
- 카드 해석 대화창
- 종장 카드 융합 연출
- 전체 화면 터치 진행
- 별빛의 기록 요약/공유/이미지 저장
```

배열법별로 새로 정의할 것:

```txt
- 카드 위치 데이터
- 카드 배치 좌표
- 챕터 제목 문구
- 위치 의미 문구
- AI prompt payload
```

## 13. 다음 확장 후보

```txt
1. 정방향/역방향 카드 시스템

```

10장 배열은 모바일 배치, 리딩 길이, AI 응답 품질 부담이 크므로 나중에 검토한다.

## 14. 절대 바꾸지 말아야 할 기준

```txt
- 게임 화면은 viewport를 덮는다.
- 질문은 반드시 봉인 연출을 거친다.
- 추천 배열은 사용자에게 표시한다.
- 사용자가 직접 배열을 바꿀 수 있어야 한다.
- AI 추천 실패 시에도 진행 가능해야 한다.
- 카드는 사용자가 직접 공개한다.
- 카드 공개는 한 장씩 진행한다.
- 해석은 카드 위치 의미를 포함한다.
- 챕터/종장 연출 중 터치 스킵을 막는다.
- 대화창/본문 표시 후 3초 뒤 진행 가능하다.
- 최종 조언은 확정 예언이 아니라 가능성의 방향과 현실적 태도로 정리한다.
- 현재 최종 화면은 SummaryScene이다.
```

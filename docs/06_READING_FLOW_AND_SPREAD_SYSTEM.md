# 06. 리딩 흐름과 배열법 공통 시스템

이 문서는 `tarrot_service`에서 타로 배열법을 확장하거나 유지보수할 때 지켜야 할 **공통 연출 흐름, UI 구조, 해석 방식, 터치 규칙**을 정의한다.

목표는 카드 수만 늘리는 것이 아니다.

> 어떤 배열법을 사용하더라도, 사용자가 같은 점술관 안에서 같은 의식의 규칙을 경험하게 만든다.

## 1. 핵심 원칙

실제 타로 리딩은 카드 하나의 뜻만 읽지 않는다.

```txt
질문
  ↓
배열법
  ↓
카드 위치 의미
  ↓
카드 자체의 의미
  ↓
카드 간 관계
  ↓
전체 흐름
  ↓
현실적인 조언
```

게임에서도 이 순서를 유지한다.

현재 구현은 1장/3장/5장 배열을 지원한다. 정방향/역방향은 아직 본격 구현 전이며, 다음 확장 후보로 남겨둔다.

## 2. 현재 기본 흐름

현재 구현된 기본 흐름은 다음과 같다.

```txt
IntroScene
  ↓
QuestionScene
  ↓
질문 카테고리와 질문 문장 기반 배열 추천
  ↓
점술사가 고른 배열 표시
  ↓
질문 봉인 연출
  ↓
CardSelectScene
  ↓
배열법에 맞는 카드 수만큼 카드 공개
  ↓
ReadingScene
  ↓
카드별 챕터 해석
  ↓
종장 종합 조언
  ↓
ChatScene
```

배열법마다 달라지는 것은 다음뿐이다.

```txt
- 카드 수
- 카드 위치 이름
- 카드 위치 의미
- 카드 배치 형태
- 카드별 챕터 수
- 종장 종합 방식
```

유지되는 것은 다음이다.

```txt
- 질문 봉인
- 사용자가 직접 카드 공개
- 카드별 챕터 해석
- 챕터/종장 터치 잠금
- 종장 종합 조언
- 후속 대화
```

## 3. 현재 지원 배열법

배열법 데이터는 다음 파일에서 관리한다.

```txt
src/tarot/spreads.ts
```

현재 지원 배열:

### 오늘의 한 장

용도:

- 짧은 조언
- 오늘의 메시지
- 가볍고 빠른 리딩

위치:

```txt
1. 오늘의 메시지
```

### 상황과 조언의 세 문

용도:

- 일/돈/현실 고민
- 막힘이 있는 질문
- 해결 방향이 필요한 질문

위치:

```txt
1. 현재 상황
2. 막고 있는 것
3. 조언
```

### 시간의 세 문

용도:

- 흐름/미래/전망 질문
- 과거와 현재의 흐름이 중요한 질문

위치:

```txt
1. 과거
2. 현재
3. 미래
```

### 관계의 거울 5장

용도:

- 연애
- 인간관계
- 상대와 나의 흐름을 함께 봐야 하는 질문

위치:

```txt
1. 나의 마음
2. 상대의 흐름
3. 관계의 현재
4. 막고 있는 것
5. 조언
```

### 선택의 갈림길 5장

용도:

- 선택 고민
- 할까 말까
- 두 방향 비교

위치:

```txt
1. 선택 A
2. 선택 B
3. 숨은 변수
4. 진짜 바람
5. 선택의 조언
```

## 4. 배열 추천 규칙

QuestionScene은 카테고리와 질문 문장을 기반으로 추천 배열을 고른다.

```txt
연애 / 인간관계 카테고리
→ 관계의 거울 5장

일 / 돈 카테고리
→ 상황과 조언의 세 문

자유 질문 + 오늘/하루/짧게/한마디/메시지
→ 오늘의 한 장

자유 질문 + 선택/둘 중/할까 말까/이직/퇴사/고백/연락할까/계속할까
→ 선택의 갈림길 5장

자유 질문 + 연애/상대/마음/연락/재회/관계
→ 관계의 거울 5장

자유 질문 + 문제/막힘/불안/고민/어떻게/해결/조언
→ 상황과 조언의 세 문

자유 질문 + 미래/앞으로/흐름/결과/전망
→ 시간의 세 문
```

질문 화면에는 `점술사가 고른 배열` 패널을 표시한다.

표시 내용:

```txt
- 배열명
- 카드 수
- 카드 위치 구성
- 추천 이유
```

## 5. 씬별 공통 역할

### IntroScene

역할:

- 사용자를 점술관으로 초대한다.
- 카드 뒷면과 VFX로 의식의 시작을 보여준다.
- 배열법과 무관하게 공통으로 사용한다.

### QuestionScene

역할:

- 질문을 받는다.
- 질문 카테고리를 받는다.
- 질문 문장을 분석해 추천 배열을 고른다.
- 추천 배열을 화면에 표시한다.
- 질문 봉인 연출을 보여준다.

유지할 것:

```txt
- 질문은 반드시 봉인 연출을 거친다.
- 추천 배열은 숨은 로직으로만 두지 않고 화면에 보여준다.
- 질문 카테고리 변경 또는 질문 입력 중 추천 배열을 갱신한다.
```

### CardSelectScene

역할:

- 배열법에 필요한 카드 수만큼 카드를 뽑는다.
- 카드 뒷면을 배열 위치에 배치한다.
- 사용자가 카드를 하나씩 공개한다.
- 모든 카드가 공개되면 리딩 단계로 이동한다.

현재 카드 수별 배치:

```txt
1장: 중앙 1장
3장: 가로 3장
5장: 위 2장 + 아래 3장
```

유지할 것:

```txt
- 카드 공개의 의식적인 감각
- 중심축 180도 플립 느낌
- 발광/파동/파티클/VFX 계열
- 카드명 한글/영문 표시 원칙
- 카드 수가 늘어나도 카드 공개는 한 장씩 진행
```

### ReadingScene

역할:

- 카드별 해석을 챕터처럼 보여준다.
- 카드 위치 의미를 먼저 보여준다.
- 그 위치에 놓인 카드 의미를 질문과 연결한다.
- 마지막 종장에서 전체 흐름을 종합한다.

현재 상태:

```txt
- 1장/3장/5장 챕터 진행 지원
- spread.positions 기반 chapterTitle/shortMeaning 사용
- 카드별 대화창 자동 표시
- 대화창 표시 후 3초 뒤 터치 가능
- 종장 카드 융합 연출
- 5장 종장 fusion-card-4, fusion-card-5 보정 적용
```

유지할 것:

```txt
- 챕터 진입 연출
- 카드 크게 보여주기
- 대화창 자동 표시
- 터치 잠금 규칙
- 종장 카드 융합/종합 조언 구조
```

### ChatScene

역할:

- 리딩 결과 이후 후속 질문을 받는다.
- 선택된 배열법, 카드, 위치 의미, 종합 조언 맥락을 유지한다.

현재 후속 대화 프롬프트는 다음 정보를 유지한다.

```txt
- 원래 질문
- 리딩 요약
- 배열명
- 카드별 위치
- positionMeaning
- 카드명과 키워드
- 대화 기록
```

## 6. 공통 리딩 시퀀스

모든 배열법은 아래 시퀀스를 따른다.

### Step 1. 배열 추천

목적:

- 질문에 맞는 리딩 방식을 사용자가 인지하게 한다.

구조:

```txt
질문 카테고리 선택
  ↓
질문 입력
  ↓
추천 배열 계산
  ↓
점술사가 고른 배열 패널 표시
```

### Step 2. 질문 봉인

목적:

- 사용자의 질문을 단순 입력값이 아니라 의식의 중심으로 만든다.

연출:

```txt
- 배경 어둡게 처리
- 질문 문장이 기도문처럼 표시
- 질문이 접히거나 봉인되는 연출
- VFX 발광/링/파티클
- 선택된 배열에 맞는 카드 선택 화면으로 전환
```

### Step 3. 카드 배치

목적:

- 배열법의 구조를 시각적으로 이해시키기.

연출:

```txt
- 카드 뒷면들이 배열 위치에 등장
- 각 카드 아래 또는 주변에 위치 라벨 표시
- 사용자가 하나씩 터치해서 공개
```

### Step 4. 카드 공개

목적:

- 실제 타로에서 카드를 한 장씩 뒤집어 확인하는 긴장감 구현.

공통 연출:

```txt
- 터치한 카드가 중심축 기준 180도 플립처럼 보임
- 공개 순간 빛 폭발
- 카드 앞면 이미지 표시
- 한글명/영문명 표시
- 공개 후 해당 카드 위치 라벨 유지
```

주의:

- 카드 공개는 즉시 결과를 모두 보여주는 것이 아니라, 사용자가 한 장씩 확인하는 의식이어야 한다.
- 카드가 많아져도 공개 방식은 동일해야 한다.

### Step 5. 카드별 챕터 해석

목적:

- 카드 의미를 단순 목록이 아니라 서사처럼 전달한다.

공통 구조:

```txt
챕터 제목 등장
  ↓
위치 의미/속삭임 표시
  ↓
카드 등장
  ↓
카드명 표시
  ↓
점술사 대화창 자동 표시
  ↓
3초 후 터치 가능
  ↓
다음 카드 챕터로 이동
```

### Step 6. 종장 종합 조언

목적:

- 카드들을 하나씩 해석한 뒤, 전체 흐름을 하나의 메시지로 묶는다.

공통 구조:

```txt
종장 제목 등장
  ↓
선택된 카드들이 빛으로 모이는 연출
  ↓
종합 조언 문장들이 한 줄씩 페이드인
  ↓
모든 문장 표시 후 3초 뒤 터치 가능
  ↓
후속 대화 화면으로 이동
```

종합 조언은 단순 요약이 아니다.

반드시 포함할 것:

```txt
- 질문에 대한 전체 흐름
- 카드 간 연결
- 배열법의 구조적 의미
- 사용자가 지금 취할 수 있는 현실적 태도
- 확정 예언이 아닌 가능성의 방향
```

## 7. 터치 잠금 규칙

리딩 UX에서 가장 중요한 공통 규칙이다.

### 카드별 챕터

```txt
챕터 화면 진입
  → 터치 잠금
제목/카드 연출 진행
  → 터치 잠금 유지
대화창 자동 표시
  → 터치 잠금 유지
대화창 표시 후 3초 경과
  → 터치 가능
터치
  → 다음 챕터 또는 종장
```

### 종장

```txt
종장 화면 진입
  → 터치 잠금
카드 융합/본문 페이드인
  → 터치 잠금 유지
모든 문장 표시 후 3초 경과
  → 터치 가능
터치
  → ChatScene
```

목적:

- 사용자가 연출을 실수로 스킵하지 않게 한다.
- 카드가 하나의 장면처럼 읽히게 한다.
- 게임 챕터 연출 같은 호흡을 유지한다.

## 8. 배열법 데이터 구조

배열법은 코드에서 데이터로 정의한다.

현재 타입:

```ts
export type TarotSpread = {
  id: string;
  name: string;
  subtitle: string;
  description: string;
  recommendedFor: ReadingCategory[];
  cardsToDraw: number;
  positions: TarotSpreadPosition[];
};

export type TarotSpreadPosition = {
  id: string;
  label: string;
  chapterTitle: string;
  shortMeaning: string;
  promptMeaning: string;
  aura?: "past-aura" | "present-aura" | "future-aura";
  layoutHint?: {
    x: number;
    y: number;
  };
};
```

AI 해석을 위해 반드시 필요한 정보:

```txt
- position id
- 화면 표시용 위치 이름
- 챕터 제목
- 짧은 의미
- 프롬프트용 상세 의미
```

## 9. AI 해석 순서

AI는 각 카드를 다음 순서로 해석해야 한다.

```txt
1. 이 카드는 어떤 위치에 놓였는가?
2. 그 위치는 질문에서 무엇을 의미하는가?
3. 이 카드의 기본 상징은 무엇인가?
4. 사용자의 질문과 연결하면 어떤 의미인가?
5. 카드 간 흐름 속에서 어떤 역할인가?
6. 사용자가 지금 취할 수 있는 현실적 태도는 무엇인가?
```

정방향/역방향은 아직 구현 전이다. 향후 확장 시 다음을 추가한다.

```txt
정방향: 카드의 에너지가 비교적 자연스럽게 드러남
역방향: 막힘, 지연, 내면화, 왜곡으로 드러남. 나쁜 카드로 단정하지 않음
```

## 10. 카드 간 관계 해석

종장에서는 반드시 카드 간 관계를 본다.

확인할 것:

```txt
- 메이저 아르카나가 많은가?
- 같은 수트가 반복되는가?
- 숫자 흐름이 이어지는가?
- 특정 카드가 전체 분위기를 압도하는가?
- 3장 흐름에서 앞/현재/뒤의 연결이 자연스러운가?
- 관계형 배열에서는 나와 상대의 카드가 충돌하는가, 보완하는가?
- 선택형 배열에서는 두 선택의 에너지가 어떻게 다른가?
- 5장 배열에서는 장애물과 조언 카드가 앞선 카드들을 어떻게 정리하는가?
```

예시:

```txt
Tower + Star
→ 무너짐 이후의 회복

Moon + Eight of Swords
→ 불안과 생각의 감옥

Death + Ace of Wands
→ 끝난 뒤 새롭게 시작되는 힘
```

## 11. AI 응답 길이 기준

현재 reading prompt에는 배열 크기별 길이 가이드가 있다.

```txt
1장
- 카드 해석 2~3문장
- advice 2문장

3장
- 카드별 2~3문장
- advice 3~4문장

5장
- 카드별 2문장 중심, 최대 3문장
- advice 4~5문장
- advice에서 카드 하나씩 재나열하지 말고 전체 구조와 카드 간 관계를 종합
```

## 12. UI 재사용 원칙

새 배열법을 만들 때 새 화면을 무조건 만들지 않는다.

우선 재사용할 것:

```txt
- 질문 봉인 연출
```

## 14. 절대 바꾸지 말아야 할 기준

배열법 확장 중에도 아래 기준은 유지한다.

```txt
- 게임 화면은 viewport를 덮는다.
- 질문은 반드시 봉인 연출을 거친다.
- 추천 배열은 사용자에게 표시한다.
- 카드는 사용자가 직접 공개한다.
- 카드 공개는 한 장씩 진행한다.
- 해석은 카드 위치 의미를 포함한다.
- 챕터/종장 연출 중 터치 스킵을 막는다.
- 대화창/본문 표시 후 3초 뒤 진행 가능하다.
- 최종 조언은 확정 예언이 아니라 가능성의 방향과 현실적 태도로 정리한다.
```

# 01. 제품 기획서

이 문서는 `tarrot_service`의 제품 방향, 사용자 경험, 핵심 기능, 화면 흐름을 정의한다.

## 1. 한 줄 정의

**신비로운 판타지 게임 속 점술관에서, 사용자가 직접 타로 카드를 뽑고 AI 점술사와 대화하며 해석을 받는 웹 게임형 타로 서비스.**

## 2. 목표 경험

사용자는 단순히 운세 텍스트를 읽는 것이 아니라, 작은 게임 이벤트에 참여하는 느낌을 받아야 한다.

목표 감정:

- “입장했다”
- “카드를 고르고 있다”
- “점술사가 나에게 말하고 있다”
- “내 질문에 맞춘 해석을 받고 있다”
- “조금 더 물어보고 싶다”

## 3. 핵심 콘셉트

### 세계관

사용자는 어두운 숲 끝에 있는 신비로운 점술관, 혹은 별빛이 모이는 아르카나 성소에 들어온다.

그곳에는 운명을 읽는 AI 점술사 NPC가 있다.

점술사는 사용자의 질문을 듣고, 사용자가 직접 선택한 카드를 통해 조언한다.

### 서비스 톤

- 신비롭지만 무섭지 않다.
- 판타지스럽지만 과장되지 않는다.
- 게임처럼 몰입감 있지만 사용은 간단하다.
- 조언은 감성적이되 현실적이다.

## 4. 핵심 사용자 플로우

```txt
1. 인트로 화면
2. 점술사 NPC 인사
3. 질문 입력
4. 스프레드 선택
5. 카드 선택
6. 카드 공개 애니메이션
7. AI 리딩 결과 출력
8. 후속 질문 대화
9. 다시 점치기 또는 공유
```

## 5. MVP 범위

첫 번째 버전은 작게 만들되, 분위기를 강하게 가져간다.

### 반드시 포함

- 시작 화면
- 질문 입력
- 3장 카드 스프레드
- 카드 3장 랜덤 추첨
- 카드 뒤집기 애니메이션
- Cloudflare Workers AI 기반 리딩 생성
- 결과 패널
- 후속 대화 입력
- 모바일 대응

### MVP에서 제외

- 로그인
- 결제
- 리딩 기록 저장
- 카드 이미지 에셋 대량 제작
- 복잡한 스프레드
- 커뮤니티 기능
- 관리자 페이지

## 6. 주요 화면

### 6.1 인트로 화면

목적:

- 사용자를 신비로운 공간에 입장시키기
- 서비스의 분위기를 즉시 전달하기

구성:

- 어두운 보라색 배경
- 중앙 마법진 또는 카드 문양
- 떠다니는 빛 입자
- 타이틀: Arcana Gate 또는 확정 서비스명
- 시작 버튼: “운명의 문 열기”

### 6.2 질문 입력 화면

목적:

- 사용자가 타로에게 묻고 싶은 내용을 정리하게 하기

구성:

- 점술사 대화창
- 질문 입력창
- 질문 카테고리 버튼

카테고리 예시:

- 오늘의 흐름
- 연애
- 일과 커리어
- 돈과 기회
- 인간관계
- 자유 질문

### 6.3 카드 선택 화면

목적:

- 사용자가 직접 카드를 고르는 느낌을 만들기

구성:

- 뒤집힌 카드 3장 또는 여러 장 중 3장 선택
- 카드 hover/touch 시 빛 효과
- 선택 시 카드가 살짝 떠오르는 연출
- 선택 완료 후 “카드 공개” 버튼

MVP에서는 3장 스프레드 고정으로 시작한다.

```txt
1번 카드: 과거
2번 카드: 현재
3번 카드: 미래
```

### 6.4 카드 공개 화면

목적:

- 결과를 바로 보여주지 않고 기대감을 만든다.

연출:

- 카드 뒤집기
- 카드 이름 등장
- 짧은 파티클
- 화면 흔들림은 과하지 않게 사용

### 6.5 리딩 결과 화면

목적:

- AI 해석을 카드별로 읽기 쉽게 보여주기

구성:

- 리딩 제목
- 요약
- 카드별 해석
- 종합 조언
- 점술사 NPC 한마디

### 6.6 후속 대화 화면

목적:

- 사용자가 해석에 대해 더 물어볼 수 있게 하기

예시 질문:

- “좀 더 현실적으로 말해줘.”
- “연애 관점으로 다시 설명해줘.”
- “내가 지금 해야 할 행동은 뭐야?”
- “긍정적인 부분과 조심할 부분을 나눠줘.”

## 7. 타로 스프레드 계획

### MVP

#### 과거 / 현재 / 미래 3장

가장 이해하기 쉽고, UI로 구현하기 좋다.

```txt
과거: 지금 상황에 영향을 준 흐름
현재: 현재의 핵심 에너지
미래: 앞으로 열릴 가능성
```

### 확장 버전

#### 오늘의 한 장

간단한 일일 운세.

#### 선택의 갈림길

A 선택과 B 선택을 비교.

#### 연애의 온도

상대와 나, 관계의 흐름, 조언.

#### 일과 기회의 문

현재 역량, 장애물, 기회, 다음 행동.

## 8. AI 점술사 캐릭터

### 역할

- 타로 해석자
- 게임 속 NPC
- 부드러운 조언자
- 사용자의 감정을 정리해주는 안내자

### 말투

- 신비로운 은유 사용
- 짧은 문장과 긴 문장을 적절히 섞음
- 과도하게 고풍스럽지 않게 현대적인 조언 포함

예시:

```txt
카드는 당신이 이미 답을 느끼고 있었지만, 아직 말로 붙잡지 못했다는 것을 보여줍니다.
지금 필요한 것은 큰 결심보다 작은 확인입니다.
```

## 10. 사운드 방향

MVP에서는 필수는 아니지만 나중에 넣으면 몰입감이 크게 오른다.

권장 사운드:

- 카드 hover: 아주 작은 반짝임
- 카드 선택: 낮고 부드러운 클릭
- 카드 공개: 마법음
- 결과 등장: 짧은 차임
- 배경음: 느리고 몽환적인 루프

## 11. 성공 기준

MVP 성공 기준:

- 사용자가 첫 화면에서 게임 같은 느낌을 받는다.
- 질문 입력부터 결과까지 막힘 없이 진행된다.
- AI 응답이 타로 서비스에 어울리는 톤을 가진다.
- 모바일에서 사용 가능하다.
- 코드 구조가 확장 가능하다.

정성적 성공 기준:

> 사용자가 “그냥 챗봇이 아니라 작은 타로 게임 같다”고 느끼는 것.

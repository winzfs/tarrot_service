# 01. 제품 기획서

이 문서는 `tarrot_service`의 제품 방향, 사용자 경험, 핵심 기능, 화면 흐름을 정의한다.

## 1. 한 줄 정의

**신비로운 판타지 게임 속 점술관에서, 사용자가 직접 타로 카드를 뽑고 AI 점술사와 대화하며 해석을 받는 웹 게임형 타로 서비스.**

## 2. 현재 제품 상태

현재 서비스는 MVP 기반이 구현된 상태다.

현재 사용자는 다음 흐름을 실제로 경험할 수 있다.

```txt
인트로 진입
  ↓
질문 입력
  ↓
카드 3장 공개
  ↓
AI 카드별 해석
  ↓
종합 조언
  ↓
점술사와 후속 대화
```

현재 제품은 “기능 검증 단계”를 넘어, **게임처럼 보이고 느껴지는 타로 경험의 완성도를 다듬는 단계**다.

## 3. 목표 경험

사용자는 단순히 운세 텍스트를 읽는 것이 아니라, 작은 게임 이벤트에 참여하는 느낌을 받아야 한다.

목표 감정:

- “입장했다”
- “질문을 의식처럼 남기고 있다”
- “카드가 열리는 순간을 기다리고 있다”
- “점술사가 내 질문에 맞춰 말하고 있다”
- “한 장씩 해석을 듣고 있다”
- “조금 더 물어보고 싶다”

## 4. 핵심 콘셉트

### 세계관

사용자는 별빛이 모이는 아르카나 성소, 혹은 신비로운 점술관에 들어온다.

그곳에는 운명을 읽는 AI 점술사 NPC가 있다.

점술사는 사용자의 질문을 듣고, 사용자가 공개한 카드를 통해 조언한다.

### 서비스 톤

- 신비롭지만 무섭지 않다.
- 판타지스럽지만 과장되지 않는다.
- 게임처럼 몰입감 있지만 사용은 간단하다.
- 조언은 감성적이되 현실적이다.
- 타로는 예언이 아니라 자기성찰의 도구다.

## 5. 핵심 사용자 플로우

```txt
1. 인트로 화면
2. 질문 입력
3. 질문 카테고리 선택
4. 과거/현재/미래 카드 공개
5. 카드별 챕터 해석
6. 종합 조언
7. 후속 질문 대화
8. 다시 점치기 또는 추가 확장 기능
```

## 6. MVP 범위와 현재 달성 상태

### 구현 완료

- 시작 화면
- 질문 입력
- 질문 카테고리
- 3장 카드 스프레드
- 전체 78장 타로 덱 랜덤 추첨
- 실제 카드 이미지 표시
- 카드 뒤집기 애니메이션
- 카드 공개 발광/파티클 효과
- Cloudflare Workers AI 기반 리딩 생성
- 카드별 단계형 리딩 결과
- 종합 조언
- 후속 대화 입력
- 모바일 대응 기반
- Cloudflare Workers 배포

### 계속 개선 중

- 카드 이미지와 카드명 레이아웃
- 모바일 화면별 여백
- 후속 대화 입력창 하단 잘림
- AI 응답 품질
- 카드 공개 연출 속도와 감각

### MVP에서 여전히 제외

- 로그인
- 결제
- 리딩 기록 저장
- 관리자 페이지
- 카드 도감
- 공유 이미지
- 사운드

## 7. 주요 화면

### 7.1 인트로 화면

목적:

- 사용자를 신비로운 공간에 입장시키기
- 서비스의 분위기를 즉시 전달하기

구성:

- 어두운 보라색 배경
- 중앙 마법진 또는 카드 문양
- 떠다니는 빛 입자
- 타이틀: Arcana Gate
- 시작 버튼: “운명의 문 열기”

현재 상태:

- 구현 완료
- 모바일에서 타이틀/버튼 크기 최종 조정 여지 있음

### 7.2 질문 입력 화면

목적:

- 사용자가 타로에게 묻고 싶은 내용을 정리하게 하기

구성:

- 점술사 대화창
- 질문 카테고리 버튼
- 질문 입력창
- 카드 펼칠 준비 버튼

카테고리:

- 연애
- 일과 커리어
- 돈과 기회
- 인간관계
- 자유 질문

현재 상태:

- 구현 완료
- 질문창 터치와 스크롤 개선 완료
- 모바일 기종별 QA 필요

### 7.3 카드 선택/공개 화면

목적:

- 사용자가 직접 카드를 여는 느낌을 만들기

구성:

- 과거/현재/미래 3장 카드
- 카드 뒷면
- 터치 시 중심축 플립
- 밝은 발광과 파동
- 카드 앞면 이미지 표시
- 한글명/영문명 표시

현재 레이아웃 원칙:

```txt
한글이름
영어이름
카드 이미지
```

현재 상태:

- 구현 완료
- 카드 이름 위치와 가독성 조정 중

### 7.4 카드별 리딩 화면

목적:

- 결과를 한 번에 던지지 않고, 카드 한 장씩 점술사에게 듣는 경험을 제공한다.

구성:

- `과거의 카드`, `현재의 카드`, `미래의 카드` 챕터 문구
- 큰 카드 이미지
- 한글명
- 영어명
- 터치 후 점술사 해석 대화창
- 터치로 다음 카드 진행

현재 상태:

- 구현 완료
- 카드와 대화창 사이 여백 조정 중

### 7.5 종합 조언 화면

목적:

- 세 장의 카드를 바탕으로 사용자의 질문에 대한 전체 흐름을 정리한다.

구성:

- 종합 조언 제목
- 조언 문장 단위 페이드인
- 중앙 정렬
- 점술사 말투 유지

현재 상태:

- 구현 완료
- 문장 줄바꿈과 가독성 QA 필요

### 7.6 후속 대화 화면

목적:

- 사용자가 해석에 대해 더 물어볼 수 있게 한다.

예시 질문:

- “좀 더 현실적으로 말해줘.”
- “연애 관점으로 다시 설명해줘.”
- “내가 지금 해야 할 행동은 뭐야?”
- “긍정적인 부분과 조심할 부분을 나눠줘.”

현재 상태:

- 구현 완료
- 모바일 하단 입력창 잘림 QA 필요

## 8. 타로 스프레드 계획

### 현재 구현

#### 과거 / 현재 / 미래 3장

```txt
과거: 지금 상황에 영향을 준 흐름
현재: 현재의 핵심 에너지
미래: 앞으로 열릴 가능성
```

### 확장 후보

#### 오늘의 한 장

간단한 일일 운세.

#### 선택의 갈림길

A 선택과 B 선택을 비교.

#### 연애의 온도

상대와 나, 관계의 흐름, 조언.

#### 일과 기회의 문

현재 역량, 장애물, 기회, 다음 행동.

## 9. AI 점술사 캐릭터

### 역할

- 타로 해석자
- 게임 속 NPC
- 부드러운 조언자
- 사용자의 감정을 정리해주는 안내자

### 말투

- 신비로운 은유 사용
- 너무 고풍스럽지 않은 현대적인 조언
- 카드의 일반 의미가 아니라 사용자의 질문과 연결된 해석
- 확정적 예언 대신 가능성과 선택지 제시

예시:

```txt
이 카드는 당신이 이미 답을 느끼고 있었지만, 아직 말로 붙잡지 못했다는 것을 보여줍니다.
지금 필요한 것은 큰 결심보다 작은 확인입니다.
```

## 11. 사운드 방향

MVP에서는 아직 제외되어 있다.

추가 시 우선순위:

- 카드 hover/touch: 작은 반짝임
- 카드 공개: 마법음
- 결과 등장: 짧은 차임
- 배경음: 느리고 몽환적인 루프

## 12. 성공 기준

MVP 성공 기준:

- 사용자가 첫 화면에서 게임 같은 느낌을 받는다.
- 질문 입력부터 결과까지 막힘 없이 진행된다.
- 실제 카드 이미지가 나온다.
- AI 응답이 사용자의 질문을 반영한다.
- 모바일에서 사용할 수 있다.
- 후속 대화를 이어갈 수 있다.

정성적 성공 기준:

> 사용자가 “그냥 챗봇이 아니라 작은 타로 게임 같다”고 느끼는 것.
